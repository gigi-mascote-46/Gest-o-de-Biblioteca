🧭 Arquitetura do Projeto – Gestão Biblioteca
🔝 1. UI (Interface de Utilizador)
Tipo: Consola

Componentes: UIController, Menu, MenuItem, EmprestimoUI, LivroUI, etc.

Responsabilidade: Interação com o utilizador (input/output), sem lógica de negócio.

🧠 2. Aplicação
Componentes: AppServiceEmprestimos, AppServiceLivros, etc.

Responsabilidade: Orquestra os casos de uso, chamando os serviços de domínio e repositórios.

Não contém lógica de negócio complexa.

🧬 3. Domínio
Componentes: GestorDeEmprestimos, CadastroDeLivros, Role (enum), exceções personalizadas

Responsabilidade: Regras de negócio, políticas, entidades e interfaces abstratas.

💾 4. Infraestrutura
Componentes: RepositorioLivrosSQLite, UnidadeDeTrabalhoSQLite, FabricaDeConexaoSQLite

Responsabilidade: Acesso à base de dados, implementação de repositórios, integração com Dapper/SQLite.
